# FinanceAPI

A multi-user finance tracking REST API built with .NET 10, Dapper and SQLite.

## Features

- JWT Bearer authentication + API Key authentication (dual auth)
- Role-based authorization (`Admin` / `User`)
- Income & expense transaction tracking
- Categorization with custom colors and sort order
- Monthly and category-based statistics
- Swagger UI for interactive testing

## Tech Stack

- **.NET 10** – ASP.NET Core Web API
- **Dapper** – lightweight ORM
- **SQLite** – file-based database (easily swappable to PostgreSQL)
- **BCrypt.Net-Next** – password hashing (work factor 12)
- **JWT Bearer** – stateless authentication
- **Swashbuckle** – Swagger / OpenAPI

## Getting Started

### Prerequisites

- [.NET 10 SDK](https://dotnet.microsoft.com/download)

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/ChristianDev87/FinanceAPI.git
cd FinanceAPI/FinanceAPI

# 2. Create your local config from the example
cp appsettings.example.json appsettings.json
```

Open `appsettings.json` and fill in the required values:

```json
"JwtSettings": {
  "SecretKey": "YOUR-RANDOM-SECRET-MIN-32-CHARACTERS"
}
```

```bash
# 3. Run the API
dotnet run
```

The database (`finance.db`) and all tables are created automatically on first start.

### Swagger UI

```
https://localhost:7185/swagger
```

## Configuration (`appsettings.json`)

| Key | Description |
|-----|-------------|
| `Kestrel.Endpoints.Http.Url` | HTTP port |
| `Kestrel.Endpoints.Https.Url` | HTTPS port |
| `ConnectionStrings.DefaultConnection` | SQLite connection string |
| `JwtSettings.SecretKey` | **Required.** Min. 32 characters, keep secret |
| `JwtSettings.ExpirationHours` | JWT lifetime in hours (default: 24) |
| `CorsSettings.AllowedOrigins` | Allowed frontend origins |
| `DefaultCategories` | Categories assigned to every new user on registration |

## API Overview

### Auth
| Method | Route | Auth | Description |
|--------|-------|------|-------------|
| POST | `/api/auth/register` | – | Register new user, returns JWT |
| POST | `/api/auth/login` | – | Login, returns JWT |

### Users *(Admin only)*
| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/users` | List all users |
| GET | `/api/users/{id}` | Get user by ID |
| PUT | `/api/users/{id}` | Update user (username, email, role) |
| DELETE | `/api/users/{id}` | Delete user |
| POST | `/api/users/{id}/apikeys` | Generate API key for user |
| GET | `/api/users/{id}/apikeys` | List API keys of user |
| DELETE | `/api/users/{id}/apikeys/{keyId}` | Revoke API key |

### Categories
| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/categories` | List own categories |
| POST | `/api/categories` | Create category |
| PUT | `/api/categories/{id}` | Update category |
| DELETE | `/api/categories/{id}` | Delete category (blocked if transactions exist) |
| PUT | `/api/categories/reorder` | Update sort order |

### Transactions
| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/transactions` | List transactions (filter: `month`, `year`, `categoryId`, `type`) |
| GET | `/api/transactions/{id}` | Get transaction by ID |
| POST | `/api/transactions` | Create transaction |
| PUT | `/api/transactions/{id}` | Update transaction |
| DELETE | `/api/transactions/{id}` | Delete transaction |

### Statistics
| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/statistics/monthly?year={y}` | Income/expense per month |
| GET | `/api/statistics/categories?month={m}&year={y}&type={t}` | Totals grouped by category |

## Authentication

### JWT (Browser / Frontend)
After login, include the token in every request:
```
Authorization: Bearer <token>
```

### API Key (Scripts / Integrations)
API keys are generated by an Admin via `POST /api/users/{id}/apikeys`. The plaintext key is returned **once** — store it securely.

Pass the key as a query parameter:
```
GET /api/transactions?apiKey=<key>
```

Only the SHA-256 hash of the key is stored in the database.

## Roles

| Role | Assigned by | Access |
|------|-------------|--------|
| `User` | Default on registration | Own categories, transactions, statistics |
| `Admin` | Admin via `PUT /api/users/{id}` | Everything + user management |

> Role changes take effect immediately for API key auth.
> For JWT, the user must log in again to receive an updated token.

## Database

SQLite is used by default. The schema and seed data (roles, default categories) are applied automatically on startup via `DatabaseInitializer`.

### Switching to PostgreSQL

1. Implement `IDbConnectionFactory` using `Npgsql`
2. Replace the following SQLite-specific expressions:

| SQLite | PostgreSQL |
|--------|------------|
| `AUTOINCREMENT` | `SERIAL` |
| `last_insert_rowid()` | `RETURNING Id` |
| `strftime('%m', Date)` | `EXTRACT(MONTH FROM Date::date)::int` |
| `strftime('%Y', Date)` | `EXTRACT(YEAR FROM Date::date)::int` |
| `datetime('now')` | `NOW()` |
